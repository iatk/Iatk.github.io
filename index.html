解析漏洞

1、IIS6.0解析漏洞
①目录分析
形式：www.xxx.com/xx.asp/xx.jpg
原理：服务器默认会把xx.asp目录下的文件都会解析成asp文件。
②文件解析
形式：www.xxx.com/xx.asp;xx.jpg
原理：服务器默认不解析;号后面的内容，因此xx.asp;xx.jpg便被解析成asp文件。

2、Apache 1.x 2.x解析漏洞
漏洞原理：
Apache解析文件的规则是从右到左开始判断解析，如果后缀名为不可识别文件解析，就再往左判断，比如test.php.a.b的”.a”和”.b”这两种后缀是Apache不可识别解析，Apache就会把test.php.a.b解析成test.php。
漏洞形式：www.test.com/test.php.a
其余配置问题导致漏洞：
(1)如果在Apache的conf里有这样一行配置 AddHandler php5-script.php 这时只要文件名里包含.php，即使文件名是 test.php.jpg 也会以php来执行。
(2)如果在Apache的conf里有这样一行配置 AddType Application/x-httpd-php.jpg 即使扩展名是jpg，一样能以php方式执行。

3、Nginx解析漏洞
漏洞原理：
Nginx拿到文件路径（更专业的说法是URI）/test.jpg/test.php后，看到后缀是php，便认为该文件是php文件，转交给php处理。php看到/test.jpg/test.php不存在，便删去最后的test.php，又看到/test.jpg存在，便把/test.jpg当成要执行的文件了，又因为后缀为.jpg，php认为这不是php文件，于是返回“Access.denued”。
这其中涉及到php的一个选项：cgi.fix.pathinfo，该值默认为1，表示开启。开启这一选项PHP可以对文件路径进行“修理”。举个例子，当php遇到文件路径“/1.jpg/2.txt/3.php”是，若“/1.jpg/2.txt/3.php”不存在，则会去掉最后的“/3.php”，然后判断“/1.jpg/2.txt”是否存在，若存在，则把“/1.jpg/.2txt”
当作文件“/1.jpg/2.txt/3.php”，若“/1.jpg/.2txt”仍不存在，则继续去掉“/.2txt”，以此类推。
漏洞形式：www.xxx.com/UploadFiles/image/1.jpg/1.php
另外两种解析漏洞：
www.xxx.com/UploadFiles/image/1.jpg%00.php
www.xxx.com/UploadFiles/image/1.jpg/%20\0.php

4、IIS7.5解析漏洞
IIIS7.5的漏洞与Nginx的类似，都是由于php配置文件中，开启了cgi.fix.pathinfo，而这并不是Nginx或者IIS7.5本身的漏洞。
漏洞形式：www.xxx.com/UploadFiles/image/1.jpg/1.php

5、Windows系统解析漏洞
旧版Windows Server中存在空格和dot漏洞，类似于”a.php.”和”a.php[空格]”这样的文件名存储后会被Windows去掉点和空格，从而使得加上这两个东西可以突破过滤，成功上传，并且被当作php代码执行。
例如：
test.asp.
test.asp(空格)

test.php:1.jpg
test.php::$DATA

python2与python3在Windows中共存

前言

当电脑同时有python2和python3时，经常会出现各种令人抓狂的问题，好在终于把它解决了，特此记录一下。。。

重命名python

首先C盘分别建一个python2和python3的文件夹，然后将python2和python3分别安装至对应的文件夹内（安装过程中都要勾选“Add Python.exe to Path“选项，两个版本的python可能描述不太一样，理解就行）。
关键的步骤：将python3文件夹内的python.exe和pythonw.exe分别改名为python3.exe和pythonw3.exe，也可以python3不动，修改python2文件夹内的两个exe程序的名字，总之两个python的exe程序不重名即可。在命令行内调用python：输入python会显示python2的版本信息，输入python3会显示python3的版本信息，成功~~

重新安装pip

还有一个pip的问题同样令人头痛，解决办法：重新安装两个版本的pip。
在CMD命令框内输入命令：
python3-m pip install --upgrade pip --force-reinstall，显示Successfully installed pip-19.0.3
python2-m pip install --upgrade pip --force-reinstall，显示Successfully installed pip-19.0.3
现在通过pip2 -V 和 pip3 -V 查看两个版本的pip信息，以后只需运行pip2install XXX和pip3 install XXX即可安装各自的python包。
当然如果不想麻烦也可以使用Anaconda来解决环境问题，不过在我的电脑上启动实在太慢了索性放弃，最后感谢大佬的博客：https://www.cnblogs.com/thunderLL/p/6643022.html

SQL注入

SQL注入漏洞原理：

①有一个参数用户可控；
②服务器对该参数没有过滤或者过滤不严谨。
漏洞本质：将用户提交的数据当做代码执行，违背了“数据和代码分离”的原则。

万能密码原理

后台登陆语句为：
SELECT * FROM admin WHERE Username = 'user' and Password = 'pass'
由于只要结果不为空就返回真，也就是登陆成功。
因此，输入万能密码or 1=1#
构造后的语句拼接为：
SELECT * FROM admin WHERE Username = 'user' or 1=1# and Password = 'pass'
通过注释符#注释后面的语句，在or 1=1的基础结果为真恒成立

二阶注入

定义
为了预防SQL注入攻击，而将输入到应用程序中的某些数据进行了“转义（escape）”，但是这些数据却又在“未被转义（Unescaped）”的查询窗体中重复使用。此时，攻击者可能注入的是一个PAYLOAD，这样就会构成一个SQL查询语句并被执行，这就是所谓的二阶SQL注入。

原理
Second-order SQL injection
当用户提供的数据由应用程序存储并随后以不安全的方式合并到SQL查询中时，会出现二阶SQL注入。要检测漏洞，通常需要在一个位置提交合适的数据，然后使用其他一些以不安全的方式处理数据的应用程序功能

特点
普通注入

在http后面构造语句，是立即直接生效的
一次注入很容易被扫描工具扫描到
二阶注入

先构造语句（有被转义字符的语句）
我们构造的恶意语句存入数据库
第二次构造语句（结合前面已经存入数据库的语句，成功。因为系统没有对已经存入数据库的数据做检查）
二次注入更加难以被发现
利用方式
攻击者在http请求中提交恶意输入
恶意输入保存在数据库中
攻击者提交第二次http请求
为处理第二次http请求，程序在检索存储在数据库中的恶意输入，构造SQL语句
若攻击成功，在第二次请求相应中返回结果

apt-get remove与apt-get purge

最近硬盘告急，仔细观察发现是虚拟机搞得鬼，开机之后硬盘容量就一直变小变小，最后只有400多M了。。。折腾了大半天也没搞出个所以来，最后只能含泪恢复快照，心疼搭了那么久的VPN。。。等以后研究明白再更新吧。不过在折腾过程中发现了两个相似的命令：apt-get remove与apt-get purge

这两个命令可以说基本一样，但是还是有细微的差别，直接划重点：
apt-get remove 会删除软件包而保留软件的配置文件
apt-get purge 会同时清除软件包和软件包的配置文件
所以如果想要彻底删除软件包，就用 apt-get purge 吧~

MySQL中的substr函数与PHP中的substr函数

一

MySQL–substr()
MySQL中的substr()函数是用来截取数据库某一列字段中的一部分
常用方式是：
substr(str,pos);
就是从pos开始的位置，一直截取到最后。
substr(str,pos,len);
这种表示的意思是从pos开始的位置，截取len个字符（空白也算字符）。
需要注意的是：如果pos为1（而不是0），表示从第一个开始。因为数据库的记录都是从1开始而没有从0开始
博文原地址：https://www.cnblogs.com/quietwalk/p/6478900.html

二

PHPsubstr()
substr(string,start,length)
参数：
1、string即你要截取的字符串
2、start即要截取的开始位置（0表示从前往后数第一个字符开始，负数表示从后往前数）
举个栗子：length=3，表示截取三个长度；length=-2，即为去掉末尾的两个字符
示例：
A echo substr("hello world",-10,-2);
输出： ello wor
解释：-10表示从前往后的第十个字符开始截取，即从字母e开始截取(空格也算字符)；
-2表示去掉末尾两个字符即去掉’ld’；
B echo substr("hello world",6,6);
输出： world
解释：6表示从第七个字符开始截取(大白话就是从头开始数六个字符然后截取)；
6表示截取6个长度；
最后一句话解释substr函数用法：
<?php echo substr("Hello world!",6); 输出world! ,只有两个参数时，第一个参数表示字符串，第二个参数表示，从第6个开始，后面的全部截取?>
<?php echo substr("Hello world!",6,4); 输出worl，有三个参数时，第二个参数表示起始位置，第三个参数为一共截取的长度?>
博文原地址：https://www.jb51.net/article/128898.htm
